# fbb_ws63 C SDK peripheral-sample fixtures for scripts/csdk-test.sh
# Each line: <elf> | <timeout seconds> | <expected UART marker (egrep regex)>
#
# Fixtures are ws63-liteos-app images built with exactly one peripheral sample
# selected; regenerate with scripts/build-csdk-samples.sh. The marker is the
# sample's own osal_printk() success line, so a match proves the peripheral model
# drove real vendor firmware correctly. These complement scripts/smoke-test.sh
# (ws63-rs Rust firmware) by exercising the peripherals from the C SDK side.
#
# elf          | to | marker  (marker must not contain '|', the field delimiter)
tcxo.elf       | 16 | tcxo get ms work normall
systick.elf    | 16 | systick get ms work normall
adc.elf        | 16 | voltage: [0-9]+ mv

# --- Known gaps (sample builds, but the model doesn't yet satisfy its path) ---
# These C SDK samples build cleanly (scripts/build-csdk-samples.sh timer dma
# watchdog) but currently hang. Root causes diagnosed (-d int / MMIO tracing);
# NOT asserted so CI stays green. (adc was fixed — see above.)
#
# A systemic fix landed first: local IRQs >=32 now auto-clear LOCIPD on delivery
# (edge/one-shot, matching the SoC's LOCIE-group + LOCIPCLR model). This killed
# the ~1M/s IRQ storms the DMA/ADC samples used to trigger; no regression
# (gpio_irq IRQ 33 still delivers per edge).
#
#   dma/watchdog - BLOCKED by the same root cause: the C SDK delivers local IRQs
#              >=32 as LOCIE *group* interrupts (mcause = one of IRQ_LOCIE0..5 =
#              26..31), and a software group handler reads LOCIPD and calls the
#              per-IRQ handler in the RAM exception table, then LOCIPCLRs. Our model
#              delivers mcause = the IRQ number directly, so e.g. the DMA-done
#              IRQ 59 never reaches hal_dma_tc_isr -> its callback never sets
#              g_dma_trans_done -> the single-transfer wait hangs. The watchdog's
#              timeout interrupt has the same issue (plus no chip_core_irq entry).
#              Real fix = deliver via the LOCIE-group model (mcause = group bit +
#              LOCIPD so the group dispatcher runs). Sizeable target/riscv rework;
#              must keep the rs per-IRQ path (gpio_irq) working too.
#   timer    - NOT a model gap. HW timer model is correct: channel 0 (IRQ 26, the
#              LiteOS systick) fires fine (~2300 IRQ-26 in 8 s). The uapi_timer
#              SOFTWARE-timer/task layer never reaches uapi_timer_start on
#              TIMER_INDEX=1 (0 MMIO writes to timer+0x200) — a LiteOS task issue.
#
# Pattern: counter-read samples (tcxo/systick) pass; completion-driven samples need
# either a deep per-peripheral state machine (adc, dma) or are blocked outside the
# peripheral model (timer = LiteOS software-timer; watchdog = no clean IRQ route).
