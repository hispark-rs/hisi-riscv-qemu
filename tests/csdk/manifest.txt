# Each line: <elf> | <timeout seconds> | <expected UART marker (egrep regex)>

# Fixtures are ws63-liteos-app images built with exactly one peripheral sample
# selected; regenerate with scripts/build-csdk-samples.sh. The marker is the
# sample's own osal_printk() success line, so a match proves the peripheral model
# drove real vendor firmware correctly. These complement scripts/smoke-test.sh
# (ws63-rs Rust firmware) by exercising the peripherals from the C SDK side.

# --- Known gaps (sample builds, but the model doesn't yet satisfy its path) ---
# Diagnosed (-d int / MMIO tracing); NOT asserted so CI stays green.
#
# IRQ-delivery note (resolved an earlier misconception): the C SDK delivers local
# IRQs >=32 by PER-IRQ mcause — the vector table sends mcause 26..31 to the LOCIE
# group handlers but mcause 32+ to a shared default_local_interrupt_handler, whose
# C dispatcher does `interrupt_number_get() = mcause & 0xFFF` and calls the per-IRQ
# handler in the RAM exception table. There is NO LOCIPCLR in that path, so the HW
# auto-clears LOCIPD on delivery. Our model matches this (mcause = IRQ number +
# auto-clear-on-delivery), so NO LOCIE-group rework is needed and there is no
# rs conflict. With that understood, the DMA sample was fixed by three concrete
# model bugs (auto-clear, int_st int_trans_st[8:15], and the src/dest_inc bit
# positions 26/27) — it now passes.
#
#   watchdog - FUNCTIONAL but not asserted. The whole WDT API is mask-ROM-resident
#              (uapi_watchdog_*/hal_watchdog_v151_*), so it is now emulated in
#              ws63_rom_call (see docs/rom-stubs.md C): enable arms a virtual one-shot
#              timer, kick re-arms it, expiry -> SoC reset. The C SDK watchdog sample
#              kicks forever -> healthy, runs to the scheduler, never resets. Its only
#              success marker is the interrupt-mode "watchdog kick timeout!" callback,
#              which is NOT modelled (a virtual-clock timer callback can't safely
#              rewrite env->pc to inject the firmware handler — not vCPU-synced), so
#              the sample has no UART marker to assert on.
#   timer    - NOT a model gap. HW timer model is correct: channel 0 (IRQ 26, the
#              LiteOS systick) fires fine (~2300 IRQ-26 in 8 s). The uapi_timer
#              SOFTWARE-timer/task layer never reaches uapi_timer_start on
#              TIMER_INDEX=1 (0 MMIO writes to timer+0x200) — a LiteOS task issue.
